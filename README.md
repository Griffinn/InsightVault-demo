🔥 InsightVaultA full-stack personal finance intelligence platform for structured data management and interactive visual analytics.📌 Project OverviewInsightVault is a comprehensive full-stack personal finance management platform designed to empower users with the tools to track, analyze, and optimize their financial behavior. Unlike traditional expense trackers that offer static lists, InsightVault treats financial data as a dynamic asset. The platform transforms raw, unorganized transaction data into high-level intelligence through a robust client-server architecture.By structuring data into intuitive categories and types (Income vs. Expense), the system enables deep-dive exploration. It leverages complex backend SQL aggregations to compute financial health metrics in real-time, converting these results into sophisticated visual representations.🎯 Objective / MotivationThe primary motivation behind InsightVault was to strip away the "magic" of modern frontend frameworks and confront the core challenges of web engineering head-on.Mastering the Fundamentals: In an era dominated by React and Vue, I chose a "fundamentals-first" approach. This project served as a rigorous laboratory for mastering manual DOM manipulation, understanding the nuances of the browser rendering engine, and implementing complex layouts using modern CSS Grid and Flexbox without the safety net of a framework.System Design & Communication: A core focus was replicating the communication patterns used in enterprise-level applications. This involved designing a custom RESTful API from scratch and managing the intricate "Request-Response" lifecycle that occurs between a decoupled frontend and backend.Data Integrity & Architecture: I aimed to practice clean architecture principles by ensuring a strict separation of concerns. The frontend acts purely as a presentation layer, while the backend maintains the "Source of Truth" through a secured relational database.⚙️ Features1. Transaction Management & PersistenceFull CRUD Lifecycle: Seamlessly add, view, update, and delete financial records.Detailed Schema: Amount, Category, Type (Income/Expense), Date, and Descriptions.Relational Persistence: Data is strictly validated and stored in SQLite/Turso, ensuring long-term data integrity and ACID compliance.2. Advanced Filtering & Data ExplorationGranular Filtering Engine: Isolate data by categories, transaction types, or keyword searches.Temporal Intelligence: Built-in date range selectors (This Week, This Month, Custom).Server-Side Pagination: Optimized for large datasets; the backend serves data in chunks, keeping the UI high-performance.3. Reports & Analytics Engine (The "Brain")Real-Time Visualizations: Dynamic pie charts and trend graphs powered by Chart.js.Aggregated Insights: Uses backend GROUP BY and SUM SQL logic for real-time totals.4. Wishlist & Savings GoalsGoal Tracking: Set targets for future purchases or savings milestones.Progress Analytics: Automatically calculates percentage achieved based on current financial standing.🏗️ System Architecture & Data FlowClient (Vercel) ➔ API (Node/Express) ➔ Database (Turso/SQLite)The Request LifecycleEvent Capture: JS captures user input (e.g., a filter change).Asynchronous Fetch: async/await fetch dispatches a request to the Express backend.API Routing: Express matches the endpoint and passes parameters to the Controller.SQL Execution: The Controller builds a dynamic SQL query to perform sorting and filtering.JSON Response: Backend returns structured data and metadata.DOM Update: Frontend receives data and re-renders the UI modules.🔌 RESTful API DocumentationEndpointMethodParams/QueryDescription/api/transactionsGETpage, limit, category, type, searchReturns paginated transactions based on active filters./api/transactionsPOSTJSON BodyValidates and creates a new transaction record./api/reports/totalsGETstartDate, endDateAggregates income/expense totals for metrics./api/reports/categoriesGETtype, startDate, endDateProvides grouped data for pie chart visualization./api/wishlistGET-Retrieves savings goals and computed progress.🛠️ Tech StackFrontend: HTML5, CSS3 (Grid/Flexbox), Vanilla JavaScript (ES6+), Bootstrap 5, Chart.js.Backend: Node.js, Express.js.Database: SQLite3 (Dev), Turso/libSQL (Prod).Hosting: Vercel (Frontend), Turso (Managed Database).🧠 Key Design Decisions1. Why No React?I chose Vanilla JavaScript to master Manual DOM Manipulation, Event Architecture, and State Management without a virtual DOM. It ensures I understand the "how" and "why" behind the tools used in modern web engineering.2. Database Migration: Turso (Cloud SQLite)Initially implemented with local SQLite, the project was migrated to Turso for production readiness. This allows for a distributed cloud database that integrates with Vercel while maintaining SQLite's lightweight nature.🚧 Project StatusThis project is currently in Active Development (Beta).Completed: Full-stack loop, SQL aggregation, analytics dashboard, and core filtering.In Progress: Fine-tuning mobile responsiveness and refactoring server paths for universal terminal execution.🔮 Future Improvements🤖 AI Predictive Analytics: Using historical data to predict future spending patterns.📊 Export Functionality: Adding PDF and CSV export for financial summaries.📱 Mobile App Wrapper: Exploring Capacitor for a native mobile experience.📂 Folder Structure├── backend/            # Express.js Server Environment
